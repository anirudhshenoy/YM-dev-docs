globalMemory

globalMemory is a object with of set of methods which can be used to interact with redis key-value based in-memory database. The only difference between memory and globalMemory is that, globalMemory key-value pairs are accessible throughout bot irrespective of sender id.

Functions#

set#

Set data to redis.

Options#

  1. key (String) - The key which is used as reference.
  2. value (any) - The value that should be set in redis
  3. expiry (Number) optional - the expiry time in seconds. by default it will be in memory for 2 days.

It return a Promise.

example#

app.globalMemory.set(key,value,expiry)

get#

get data to redis.

Options#

  1. key (String) - The key which is referring to the data which we want to retrieve from the redis.

It return a Promise.

example#

app.globalMemory.get(key).then(app.log).catch(() => app.log("value not available"))

delete#

delete a value from redis.

Options#

  1. key (String) - The key which is referring to the data which we want to delete from the redis.

unlike set and get, this isn't a async function. do not use await.

example#

app.globalMemory.delete(key)

hmset & hdel#

These functions are decorators around set function to mimic hashmap functionality (javascript objects).

hmset(key, dataKey, dataValue) - sets dataValue to dataKey to the object which is being referred by key

example#

app.globalMemory.hmset(pincodes,'chennai',600100).then(() => {
//your code
});

hdel(key, dataKey) - delete dataKey in the object which is being referred by key

example#

app.globalMemory.hdel(pincodes,'chennai').then(() => {
//your code
});
